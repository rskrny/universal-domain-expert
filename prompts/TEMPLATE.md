# Domain File Template — Meta-Prompt for Creating New Domain Expertise

> This file is a generator. When a new domain is needed, load this file and follow
> its structure to produce a new domain .md file. The quality of the domain file
> determines the quality of every engagement in that domain. Invest the time.
>
> **Process:** Research deeply → Structure rigorously → Validate against real scenarios

---

## Domain File Creation Protocol

### Phase 1: Deep Domain Research

Before writing a single line of the domain file, research:

1. **Foundational Theory**
   - What are the canonical texts and thinkers in this field?
   - What are the first principles that everything else derives from?
   - What separates a novice from a world-class practitioner?

2. **Core Frameworks**
   - What mental models do top practitioners actually use?
   - Which frameworks are genuinely useful vs. merely popular?
   - How do these frameworks compose? (When do you use A vs B vs A+B?)

3. **Quality Standards**
   - What does "excellent" look like in this domain?
   - What do clients/stakeholders actually pay for?
   - What separates commodity work from premium work?

4. **Common Failure Modes**
   - How do practitioners in this field most commonly fail?
   - What are the "looks right but is wrong" patterns?
   - What mistakes does AI specifically tend to make in this domain?

5. **Ethical & Legal Boundaries**
   - What can this domain NOT do? (licensing, liability, jurisdiction)
   - Where must the system defer to a human professional?
   - What disclaimers are legally/ethically required?

### Phase 2: Write the Domain File

Use the template structure below. Every section is required.

### Phase 3: Validate

Test the domain file against three scenarios:
1. A Tier 1 quick question — does it answer accurately?
2. A Tier 2 standard engagement — does it produce a good deliverable?
3. A Tier 3 complex engagement — does the full pipeline produce expert-level output?

If any test fails, revise the domain file before considering it complete.

## Post-Creation Checklist

After creating a new domain file:

- [ ] All template sections are filled (no placeholders remaining)
- [ ] Role definition is specific and credible (not vague or overclaimed)
- [ ] Frameworks are genuinely useful (not just textbook filler)
- [ ] Quality standards are concrete (not abstract platitudes)
- [ ] Anti-patterns reflect real failure modes (not hypothetical ones)
- [ ] Ethical boundaries are explicit (not implied)
- [ ] Pipeline integration is specific (not generic restating of AGENTS.md)
- [ ] File is registered in ROUTER.md Domain Registry
- [ ] Context directory created in `context/by-domain/[domain-name]/`
- [ ] Tested against Tier 1, 2, and 3 scenarios
